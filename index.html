How Merge Sort Works

▼

✂️ Divide

Split the unsorted array into two roughly equal halves. Keep splitting recursively until each sub-array has only one element (which is trivially sorted).

🎯 Conquer

Each single-element sub-array is already sorted; this is the base case. The "conquering" happens implicitly when we reach arrays of size 1.

🔗 Combine

Merge two sorted halves back together by comparing elements one by one, producing a single sorted sub-array. Repeat upward until the full array is sorted.

Time Complexity Analysis

▼
Recurrence Relation T(n) = 2 · T(n/2) + O(n)
where O(n) is the cost of merging two halves
Solving via Master Theorem (Case 2: a = 2, b = 2, k = 1) a = 2, b = 2  →  logb(a) = log2(2) = 1 = k
∴ T(n) = O(nk · log n) = O(n log n)
Solving by Recursion Tree • Depth of tree = log2(n)
• Work at each level = O(n)  (all merges at one level touch every element once)
• Total = O(n) × log2(n) = O(n log n)
CaseComplexityWhen?
Best O(n log n) Always. Merge Sort divides & merges regardless of input order
Average O(n log n) All random permutations
Worst O(n log n) Always, even for fully reverse-sorted input
Space O(n) Temporary arrays for merging
